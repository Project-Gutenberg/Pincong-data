很多人都在说隐私问题，我说一下技术。

最早一批淘宝用户应该是只有 web 端的，那个时候的确体验有限。

但其实你会发现，现在的 web 端淘宝和当年变化真的不大。

简单来说，我们常见的一个首页，一般有 logo，搜索框，菜单，轮播，瀑布流 / 滚动列表等等，结构上还有页眉页脚。

**而移动端需要什么？绝大多数情况下，搜索框，轮播图，瀑布流，剩下的内容一般不会同时展示给用户。**

让我自己评价，移动端的布局设计和实现比 web 端要容易太多了，因为移动端特殊的屏幕比例，导致了用户的关注点倾向于单一内容，根本不会需要你同时展示菜单轮播和滚动列表，因为用户的屏幕就这么宽。所以在今天，滚动列表和瀑布流以及其他的方式非常流行，绝大多数情况下就是两列或者一列。

但是如果你给 web 端做成这样，用户能砍死你，最小的屏幕也在 13 以上，你给人家放一列或者两列，要不然大图要不然大间距，用户体验极差，不知道还以为是老年版。

**pc 市场的缩水不是五六年了，移动端兴起也不止五六年，还是这么多用户，你不能要求用户左手手机右手鼠标，所以 web 端下滑是必然，肯定会转向移动端，而且还能降低开发和设计难度。**

我还想提一点，就是装客户端可以增加粘性。

客户端并不是完全处于搜集用户信息，因为移动端访问性能真的不好。比如各类小程序还有移动 web 端的产品，我相信没有几个体验会比客户端更优秀；另外，一个客户端装上了就在你的手机里建立起根据地，今天给你 push 一下，明天再给你 push 一下，时间长了不是你把它卸载就是它把你拥有。当然你也可以把通知自启关了，但是它也依然在你的手机里。

如此以往，粘性不久有了吗？**在以前，很多网页打开之后，都会提示你是否要将本站设为主页，是否要将本站添加到收藏夹，是否要将本站保存快捷方式到桌面上。**在我看来，这些都是增加用户粘性的方法，只是在今天，没必要了。

至于隐私问题，我没调查过不评论。**当杭州说他们跑了三头豹子，这些豹子可能已经散养了快一个月了。**

### 知乎用户 Anonymous 发表
    
这实际上是一个用户和开发者 (你们愿意把开发者说成资本家也行) 互相选择互相促进的结果, 特别是在国内市场特别明显.

说实话, 在移动端, 原生应用 (App) 的用户体验就是要比网页应用容易做好, 开发难度就是要比网页应用要低 (只要有点稍微高级点的功能). 网页应用虽然可以 "一次开发到处浏览", 但是受制于移动端浏览器的性能, 和 HTML/JavaScript/CSS 的技术限制, 要实现和原生应用一样的效果要难的多 (要实现在不同浏览器中的功能一致性也是并不容易的一件事情), 更不用说性能差距巨大. 当年 Facebook 很长一段时间里, 他们的移动 App 实际上是用的内嵌 WebView 的方式, 也就是用网页应用开发的, 但是一段时间后发现实在是有太多问题, 被迫彻底重写为原生应用 (both for iOS and Android). 对于追求更好体验的开发者和用户来说, 原生应用肯定是更好的选择, 就不说原生应用带给开发者的其他各种好处了.

原生应用相对于网页应用当然也有缺点, 最大的一个, 就是用户必须主动下载安装后才能使用. 所以大家也在想办法来解决这个问题. 比如 Apple 在 2020 年 WWDC 推出的 App Clips, 比如微信和其他很多应用提供的小程序框架, 都在力图降低用户使用应用的障碍.

但是抛弃网页的趋势在国内相对于欧美市场更加明显, 原因我个人认为是国内的网民使用的设备以手机为主导致的. 欧美早就实现了个人电脑的普及, 会日常使用 PC 的用户比例相对国内来说高很多, 所以开发者就很难真的抛弃桌面用户, 而桌面用户的习惯还是 Web 更多, 所以欧美开发者会更有动力开发 Web 应用. 而随着 Web 技术的发展, 使用响应式布局 (Responsive design) 开发的 Web 网页可以在桌面端和移动端同时拥有不错的体验, 使得移动用户自然拥有了可用的入口.

而国内用户中绝大多数其实并不会日常使用 PC, 在这种情况下形成了负激励: 用户使用 Web 的比例低, 导致开发者没有动力开发桌面 Web 应用, 而花精力专门开发移动 Web 应用带来的效益远远不如专注于原生应用来的有效, 结果是移动网页应用匮乏, 用户更加不愿意使用移动网页.

至于很多人说的隐私问题, 其实一定程度上和很多答主说的恰恰相反: 对于绝大多数普通用户, 移动端应用更容易实现隐私保护, 而 Web 端反而是更容易隐私泄露的. 移动端只要操作系统愿意, 运行在沙箱中的应用可以什么未授权隐私都拿不到, 连跨应用追踪用户都做不到 (iOS 和 MIUI 都众所周知). 而除了知道装插件来避免隐私泄露的 Geek 们, 普通用户打开一个网页, 可能同时就把自己的 GMail 账号, Amazon 账号, Facebook 账号, Twitter 账号一并交给了这个网页的开发商 (在国内, 上交的可能就是 QQ/Baidu/Weibo.. 账号), 而且毫无察觉. 这些年, 因为保护消费者法规的缘故, 众多欧美网站才开始被迫展示类似的弹窗让用户确认:

![](https://images.weserv.nl/?url=https%3A//pic4.zhimg.com/v2-7a6f48df457a3357ce3134d408116ed3_r.jpg%3Fsource%3D1940ef5c)

(还真以为在桌面端就拿不到你的电话号码电子邮件好友列表这些啊? 只是你手机里的联系人可能确实不好拿)

诚然, 对于那些对隐私关注的用户来说, 网页应用是更可控的, 更容易自己动手来阻止隐私泄露, 而原生应用只能指望操作系统和应用开发者的良心. 但是对于普罗大众, 其实是反过来的. 对于那些一众靠卖隐私为生的公司们 (比如 Facebook, Google, etc.), 如果让他们选择, 他们肯定更希望用户用 Web 更多, 至少, 省的 iOS 做一次隐私功能更新他们就鸡飞狗跳一次.

但是对于那些明明有移动网站, 但是唯一作用就是给 App 引流的那些网站 (典型例子就是新浪), 那就没话说了..
