统信软件（就是深度现在的母公司）适配中心的普通员工前来回答一发，生态建设正好是适配中心负责的范畴之一。

首先赞赏

[@面向搜索引擎编程]()

的理解，UOS 本身是一个普通的 Linux 发行版，当前产品正式名称是 统信 UOS，中文名统信操作系统，有三个版本：专业版、个人专业版和社区版，其中专业版主要面向政企用户，而非普通消费者，支持国产 6 个 CPU 平台；个人专业版主要面向中小企业和专业用户（指特定行业用户，而不是技术人员），支持 X86 和 ARM 平台；社区版也就是 deepin，主要面向爱好者和专业技术人员，仅支持 X86 平台。

大家一直在讨论的 UOS，我理解大家应该指得是狭义上的 UOS，即特指面向政企的专业版。

我浏览了不少社区的帖子，有一个感慨，大多数人将 UOS 专业版与 deepin 混为一谈，继而对标 windows（普通家用版本）。在逻辑上就是混乱的。

首先 UOS 专业版面向特定人群，聚焦国产 CPU 平台，当前阶段的主要应用场景是办公相关，与之对标的应该是 windows 政府采购版本或 windows 企业版。目前这一生态领域，基本能够做到可替代，只是大家对政企办公领域的厂商不是很了解。UOS 专业版不面向普通消费者，不太可能考虑游戏、娱乐等领域。大家也不用拿 “不能打游戏” 这个来喷，本身系统也不是干这个的。

UOS [专业版](https://www.zhihu.com/search?q=%E4%B8%93%E4%B8%9A%E7%89%88&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra=%7B%22sourceType%22%3A%22answer%22%2C%22sourceId%22%3A1079479372%7D)不也面向专业的程序员 / 技术人员做开发或研究用，限制了一些权限，本质是为了更好的维护，以及降低使用门槛，在配合特定使用场景前提下，这个策略是可行的。

其次 UOS 维护问题，这可能是部分运维朋友在问的话题，是不是没有组策略域管理就不好维护，是不是会造成大量的涉密环境下因为机器无法联网带来的维护和升级问题。UOS 维护策略如下：

1 - 经过市场调研，政企用户 90% 的场景下是可以连接公网，只是网络会做某些限制，因此系统升级不存在太大问题；针对内网用户，也提供了内网仓库搭建的方案，可以做到类似公网的应用分发和内网机器升级。至于连内网都没有的机器，恐怕是非常特定使用环境，需不需要升级是另一个话题了。

2 - 维护和管控问题，目前已经有厂商在围绕 Linux，开发终端管控和运维方案和标准，这方面比较专业，我不专业，不详述。

最后谈一谈系统开发和生态的话题，这个比较复杂。

系统不稳定？一方面员工确实少，精力有限，最近在抓紧招人，希望能尽快修改更多 bug。系统还是不稳定？是不是因为您用的 intel 或 amd 的机器，UOS 专业版没有面向这两个 CPU 做特定优化，例如休眠无法唤醒、关机问题等，实际上是主板驱动或内核对 cpu 支持的问题，NVIDIA 和 [amd 显卡](https://www.zhihu.com/search?q=amd%E6%98%BE%E5%8D%A1&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra=%7B%22sourceType%22%3A%22answer%22%2C%22sourceId%22%3A1079479372%7D)也未做很好的适配（是他们不适配 UOS，不是 UOS 不适配他们），国内显卡厂商正在努力做适配中... 如果有兴趣，可以等一下 deepin 或个人专业版，有专门优化，体验会好很多。

生态问题，跟 BAT 都对接过，大家关心的一些 2C 应用我们也谈过，厂商有厂商的顾虑；用户可能觉得他们应该开发，他们也考虑开发，但实际上面临很多具体的困难，比如到底基于哪个[发行版](https://www.zhihu.com/search?q=%E5%8F%91%E8%A1%8C%E7%89%88&search_source=Entity&hybrid_search_source=Entity&hybrid_search_extra=%7B%22sourceType%22%3A%22answer%22%2C%22sourceId%22%3A1079479372%7D)开发，怎么保证不同发行版兼容性，后续怎么维护，没有懂 Linux 开发的员工怎么办，没有机器（国产 CPU）怎么办？

Linux 的应用在厂商侧是有优先级的，比如某厂商对于开发企业级应用，特别是涉及到办公领域的 linux 服务端应用，非常积极；2C 桌面产品就不那么积极，用户量不大，利益点不多。另有某企业也在头疼，Linux 有那么多包管理机制，那么多格式，每个发行版库版本又不统一，怎么去维护，怎么保证兼容性，凭啥 windows 有几亿用户就打一次包，linux 没多少用户还得多次编译和打包。在此时 Linux 碎片化给用户带来的自由选择性优势，反而成了生态建设的劣势。

Linux 的应用在用户侧是有侧重点的，肯为 UOS 专业版买单的用户，大多关注还是在办公，因此非办公领域的应用适配就不那么受厂商青睐。特别是个人用户关注的 2C 产品，反而不是政企用户关注的。这里有一个需求的错位，即普通爱好者普遍关注的，反而不是政企关注的；政企关注的，普通用户既不了解又不想了解。

Linux 的应用在操作系统厂商侧是有苦衷的，普通爱好者是舆论的主要影响群体，但并没有提供足够的收入来源给企业；政企用户较少发声，但能够养活操作系统厂商；一来二去，就会造成操作系统厂商不重视用户需求的假命题。不是不重视，是真的精力和经费不足。连亏 10 年做社区发行版已经够情怀了吧？

为什么一定要有这么个 OS 呢？一方面世界上很多国家还遭受着 mac OS 和 windows 制裁的威胁，有一个在白宫控制范围以外的、给正常人类使用操作系统很重要；另一方面 Linux 开发者自由免费开源热情，虽然大部分软件都是粑粑，但是少量的、长期维护的、质量稳定的软件是可以满足普通人的需求的；更重要的一点是，这是给所有 ARM、RISC-V 等 ISA 笔记本、台式机解决方案的最后一个机会。

deepin 跟麒麟、红旗之类的操作系统最大的不同，就是后者基本都是白嫖由纳税人血汗钱组成的 NSF 的钱做起来的，起根上就具有严嵩蔡京贾似道的血统基因——而如我之前所说，deepin 的发家史，是蔫土匪工程师们起义下山洗白招安上岸后实现的第一桶资本原始积累，所以哪怕之后有 “采购扶持计划” 给 deepin 滋滋打钱，至少 UOS 的出身是相对清白的，也是真正一批梁山好汉们起家的。
