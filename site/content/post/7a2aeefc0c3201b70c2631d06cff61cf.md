我非常有资格回答这个问题。

我是 UI 设计师，给某一线城市的区级政府做过他们一个自用的系统。

一线城市，中心区，区委组织部。这几个关键词可以说明，除开北京中央，这就是全国素质最高的一部分公务员。

保密红线，其他信息不透露太多。

为啥这些网站既丑还难用？

你就想象一下你妈逼你穿毛线裤，但你拿着一条淘宝一百块的保暖绒裤跟你妈说。

你说
--

**妈**（领导您好）

**既便宜**（复用率高，能最大程度加快上线排期）

**还保暖**（交互方便好用，尽量减少复杂操作）

**不仅轻薄好看**（简洁明快的排版布局，庄重又合理的配色方案）

**又是大牌**（我们是专业的，这是我们内部讨论后的最佳方案）

**有保障**（我的研发团队技术您放心，有问题随时解决）

**你自己打的毛线裤**（您以前用的政务系统）

**又厚**（各项组件千奇百怪）

**还透风**（这个复杂程度不如回到纸质化办公时代）

**也不好看**（Windows XP 时代的经典设计，office 文档式的无脑布局）

**现在都没人穿这种裤子了**（您看一下我们列举的其他企业单位用的各种系统界面）

**还是不穿这条了吧**（我们是专业的，这样肯定更好）

你妈说
---

**你这孩子**（你是？哦算了不重要，我先说几点）

**尽说瞎话**（你的意思我知道，但我觉得还有几点问题）

**那工厂机器做的裤子**（你的这些方案我都看了，嗯，就这八个方案吗？还有没有更多的看一下）

**还能有我自己手打的好？**（但我们觉得，还是不行）

**卖的是便宜**（项目排期这个我们是知道的，但是首要任务，还是要做一个好系统，对不对？）

**但再便宜那不是钱呐？**（当然了，这个上线日期还是要遵守的，这个我们也是向领导汇报过的，要承担责任的）

**小孩子要什么好看**（嗯，不要搞这些花里胡哨的，我们的同志看了可能会不太喜欢，你看那个毛笔字不错，这个红色也可以，你们再做一版大红色的、毛笔字的方案我们看一下）

**穿得暖就行**（你看这不是很好嘛，你们啊，怎么自己不多想想呢，什么事都要我来问了才会做）

**听话**（不用说了）

**我再改改这裤脚**（我要拿去给领导汇报一下，领导可能会有点修改意见，你先别急）

**就穿这条**（领导的意思是不太满意，你们再做一版，但还是往我那个方向靠啊，明天就要）

### 知乎用户  吴伟​​ 发表
    
因为进化源于选择压力。

政府网站岂止 UI 差，其实交互也差、信息架构也差。很多答案都说了，想找的信息找不到，找到了读不懂。

这是为什么呢？政府网站的设计者真的看看不出来 ui 差吗？很有可能他们真看不出来。

为了解答这个问题，我先带你回忆一般的商业网站是怎么进化的。比如有两个相互竞争的商业网站 A 和 B，谁也没法从对方手里把用户抢过来。有一天 A 的老板突然想明白了，就给设计师下达任务：去研究一下用户的审美，按照大家喜欢的调调去改版。好，设计师接到任务后 996，两周之后新版本上线，ui 漂亮了，果然抢来了一部分 B 的用户。

这时候 B 老板坐不住了，于是同样的流程在 B 公司也走了一遍，两周之后 b 的 ui 也好了，于是一部分用户又回流了。

你看，经过一个月，A 和 B 的用户量可能又回到了一个月以前的平衡，谁也没从谁那里抢来用户，但是 A 和 B 的网站都比以前好看了，两个网站的设计人员的审美也都与时俱进了。这种进化在商业网站之间每天都在发生，所以商业网站就越来越好看了。这就是因为网站感受到了用户选择的压力，从而带来的审美观进化。

回过头来再看政府网站。

咱们都知道，虽然每个政府部门都有负责网站建设的官员，但是肯定不是官员自己带着研发团队做网站，对吧，他一定外包给找供应商。而对于这些主管官员来说，用户量并不是政府网站考核的 kpi，政府网站需要的是稳定、不能出错、及时体现上级要传达的精神。

所以对于主管官员来说，在选择供应商时，可不可靠成了最主要的标准。因此，主管官员倾向于选择熟悉的、知根知底的供应商，并且一旦选择了就更愿意长期合作，轻易不会更换。

结果就是供应商失去了进化的动力，很多政府网站的 ui 还保持在十几年前的水平。也就是说，你觉得政府网站丑，是因为你站在现在的审美观上；但是你要是站在十几年前的审美水平上看，这些网站的 ui 就还可以。

**“这个 logo 要放在中间的位置，“党” 字不能用蓝色，要政治过硬”**

总而言之都是在你看来无事包经的需求变更，在领导看来是一项使命，一项将来报道不能有偏差的任务。“这样设计比较丑” 的逻辑和完成任务的逻辑不能兼容，这是根源所在。甲方把项目的结果视作重要考核，其中的艺术加工只是一个常规作业，乙方也没必要过度把一项工作当成终极的艺术追求。

所以设计师们其实也应该释怀，我的目的就是交工赚钱嘛，你说怎么就怎么呗。不要去抵触。我的想法多惊艳，我的元素多写意你居然无视，这些只能让自己怄气，因为主不在乎。

4\. 视觉效果的实现问题。
--------------

同事之间存在技术栈之间的差异，硬件平台上视觉实现也可能会存在不同的障碍。例如在某 pad 上设计界面，UI 稿提供的设计语言是 “圆润”，于是你搜索栏是圆弧矩形，关闭按钮是圆形打底，弹出对话框也有一定的圆弧半径…… 可是，硬件的开发环境仅支持直角矩形的绘制，且不支持 png svg 的引用该如何呢？这是硬件和前端技术的制约。

此外，图像处理软件做出的效果会更灵活，例如阴影的实现，ps 中就有无数的参数供你选择，但是到了开发层面，因为各种原因，可能只能退而求其次了。UI 落地后，很多细节都没有了。

设计师的电脑屏幕的素质也更高，通常和业务层面的显示设备并不一致，甚至有时候生产环境下的显示器会产生严重色差，你做出来是蓝色的，呈现的时候完全有可能转变成青色或者更鬼畜的颜色。

说一句题外话，在黑白电视的时代，转播足球时，为了分清主客队，在球服上，不仅做出了纹理和设计元素的区别。而且也考虑了明度。

查一查资料，发现色盲和色弱的比重并不少，有时候 UI 设计师可以关闭几个通道或者去色界面，模拟一下最终的效果，这也算是一种人文关怀。

领导可以不重视设计师的想法，但设计师一定要重视所有用户的体验。

**后记**
------

我国的产业经济发展程度导致社会的美学逻辑现下仍然是功能、效率至上的，在增量经济转入存量经济的时代，美术设计的需求也在不断提升，所以设计师日后一定会等到自己发光的机会。

这个答案下面一堆人都没回答到点子上，或者拿自己过往的经理去尬答。

其实原因很简单，对于政府部门，一般他们的网站需求并不是好看，而是安全➕稳定，你看一些政府网站你会发现一般这些网站的风格都显得很硬，很简单就是为了防篡改，能用文字的地方就用文字绝对不用图片，很多 flash 动画人家是绝对不会用的。甚至很多敏感单位的页面上很多文字都是不可改动的。

为什么会这样呢？

因为出现过政府网站被挂上黄赌毒和反华图片的先例。政府网站往往是被国外攻击的重点。如果发生了政务网站被挂马或者篡改造成社会事件，领导是要负政治责任的。

在这种环境下，政务项目一般有个特点，本身产品的造价少，但是为了一个网站花费十几万做网站安全是很正常的事情。

所以在这种一般原则下，领导是不可能同意你把网站做的花里胡哨的，一般政务网站的要求就是简单，一是为了方便老百姓使用，找得到在哪，二就是为了减少不必要的运维成本，调试简单，三就是避免风险，保证安全稳定。

那好了，我来问问你，你接了一个政府项目，你跟领导说我搞了一个新架构可牛逼了，领导问你简不简单，安不安全，网站挂了多久能起来你怎么回答？
