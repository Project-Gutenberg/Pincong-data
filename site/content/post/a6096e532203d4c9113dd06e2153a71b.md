### 知乎用户 匿名用户 发表
    
作为曾经的 csdn 版主，给你们说点内幕吧。csdn 的彻底放飞始于 2017 年，那年 csdn 来了一位某搜索引擎跳来的高管。当你明白了手术室来了一位抡大锤的，你就知道了躺在手术台上的病人没救了。这个人唯一的经验就是搞流量。可悲的是，他不理解的是，高流量能成功是互联网跑马圈地阶段的大势所趋，增量市场本身的特点所致，而不是他的能耐。然而他却偏执地认为，这就是他可以不断获得成功的独门绝技。他来了以后，迅速搞 seo，在社区、博客任何可以插入链接的地方，都加上乱七八糟的东西。同时引入大量三俗小广告。为了帮助 csdn 节流，他耍赖把已经发放给版主的积分（相当于工资）取消，在版主的强烈抵抗下，他最终没有搞成这个。但是这波操作，让论坛几乎一半的版主走了。之后他为了刺激流量，滥发 c 币，让用户做各种 "任务"，然后如法炮制地又耍赖，搞出了 c 币清零和过期。等于说白白把用户耍了。这波操作后，又走了很多人。眼看社区不行了，搞了很多机器人账户，把 stackoverflow 等内容爬了以后翻译成中文，以及搞了所谓 “聚合页”，也就是根据用户来路搜索信息关键字，虚构了很多没有内容的页面。所以诸位知道了，无论搜索什么，都能找到 csdn，但是打开一看，什么相关内容都没有。最恶劣的是，去年年底他把问答板块前几名的全部 c 币全部冻结，毫无理由。对于他手下的员工，也是无缘无故地呵斥，掩盖其工作的无能。对待核心用户和自己的手下，就这样，你说他能对一般用户善良么？csdn 被他搞到如今，已经完全黑化了。表现为，首先是各种三俗广告，后来又大肆靠盗版资源收费，把下载积分由 5 分提高到 50 分，等于让早期上传贡献者的贡献贬值 10 倍。再后来，搞了很多虚假的课程，所谓的老师课上了一半跑路了，按理说应该退钱吧，就是不退。大家可以看看两个网站，一个叫互联网投诉举报平台，csdn 的母公司长期都是这个网站联系不上的黑户。

![](https://images.weserv.nl/?url=https%3A//pica.zhimg.com/v2-0fb89412e82f36a444fad845417c9d64_r.jpg%3Fsource%3D1940ef5c)

另一个是黑猫投诉，上面各种 csdn 坑人的骗局，绝对比看今日说法还带劲。csdn 资源抽成比例是 4:6，就这样还设置了总总规则限制提现。说白了，csdn 就是坑，就是骗。已经没救了。csdn 已经和垃圾用户深度绑定。这意味着，csdn 的用户基本都是一些信息不足，不了解互联网套路的，csdn 再想好好经营，这些用户未必买单。相反，加大套路，或许反倒能有 “业绩效果”。反过来说，csdn 已经是一个黑化的网站，稍微智力正常点的程序员。都受不了它那套，所以，低端用户绑定了 csdn，csdn 绑定了低端用户。csdn 不可能有未来了。

* * *

没想到这么多人关注我的回答。我再补充几条。首先，在这个高管的祸害下，csdn 已经彻底万劫不复了。口碑差到什么程度有目共睹，垃圾站、内容农场、low、广告站、套路多、没底线、耍赖，总之没有什么正面的观感了，它的运行模式也根本就不清楚。

![](https://images.weserv.nl/?url=https%3A//pic1.zhimg.com/v2-7b9b95d0cd16c39191e1bd9e23fb0a9d_r.jpg%3Fsource%3D1940ef5c)

简单来说，有价值的网站无非就是是否帮用户省钱或者帮用户省时间两种。两者都能做到的，就是良心网站（如图右上角）。退而求其次，也得 2 者做到其一吧。但是，csdn 的位置就很尴尬了。

最近，csdn 从微软请来了邹大神，但是在我看来其实只是借他的声望，无助于挽救 csdn 颓势，甚至邹大神会不会因为 csdn 弄个晚节不保，都很难说。csdn 的问题在于，他收割来收割去，都只是非程序员，程序员已经基本都 “-csdn” 了。因此再想提高内容质量，已经没意义了。而且有那个高管掣肘，他断断不可能提高 csdn 的质量，让正常水平以上的来 csdn 来干嘛？来了解 csdn 的黑然后让天下人都知道？

所以 csdn 现在转型做技术以外的生意了。比如下载站，现在首页打开，推荐的都是些不知道哪里找来的文档，什么农药专利、中小学文档，凡是个文档，都拿来高价卖钱。

![](https://images.weserv.nl/?url=https%3A//pic3.zhimg.com/v2-a54088f4957ea6e8797991ca820137f2_r.jpg%3Fsource%3D1940ef5c)

一个好的网站，应该带领用户成长，可是 csdn 恰恰相反，他不但自己堕落，还拐带用户一起学习 “seo”

![](https://images.weserv.nl/?url=https%3A//pic1.zhimg.com/v2-696f9ff2a1e31741482f0ff75133d0e3_r.jpg%3Fsource%3D1940ef5c)

看看这里吧，csdn 的后台创作管理界面，看看，他提供了百度收录功能，就是让每个创作者去按照如何让百度多收录的标准优化自己的创作。所以 csdn 上发布内容的都是什么人？他训练出的是什么用户？培养用户成什么？csdn 不但源源不断输出劣质内容污染搜索引擎，还在不断培养污染搜索引擎的人，这就过分了。

最后，你们也不要 at 蒋总了，csdn 已经被那个高管把持了，他欺上瞒下。蒋总呢，他早就开启了疗养模式，2017 年以后，同步从各种场合消失，你们看看他的知乎，是不是 2017 年以后没动静了。所以，2017 年以后的 csdn，已经是一个完全不同的网站了。和之前的 csdn 早就不是一回事了。

还打算继续在 csdn 创作的用户，请你们慎重。我为 csdn 贡献了 10 万个回答（不是赵 4 老师那样的回答），摸着良心我的回答都是我的见解，没有一个是复制粘贴剽窃和灌水的。最后的结果就是，csdn 把我回答问题题主花钱购买的然后给我的打赏 c 币全部冻结，我的 c 币是回答了 10 年 10 万个问题积攒起来的，所有的贡献一笔勾销。这笔仇我不会忘记。最后我想对那个高管说一句，**你迟早要还的，你欠我，欠这个社区每个曾经做出贡献，因为 csdn 名声狼藉而受伤的创作者，欠每一个付钱给你们的消费者，欠因为看了你们的诈骗广告而蒙受损失的人，欠你带领的但是没有办法成长的员工、欠被你排挤走的 csdn 员工、欠被你们盗版、抄袭的人，欠全体开发者，欠使用搜索引擎不来 csdn 网站都免不了骚扰的人，欠每一个互联网用户，你是要还的！！！**

（一）技术社区推荐
---------

### 1、Github

网站地址：[https://github.com/](https://link.zhihu.com/?target=https%3A//github.com/)

![](https://images.weserv.nl/?url=https%3A//pic1.zhimg.com/v2-976a7686423aa3da41adc08b93e40d98_r.jpg%3Fsource%3D1940ef5c)

作为程序员如何不知道这个网站的话，你可能需要回炉重造了。

我个人觉得是目前最牛逼的技术社区，**Github 上面有很多非常优秀的开源项目**，包括阿里、腾讯、字节跳动、美团、Google、Micosoft 等国内外大厂都有自己的 Github 开源库。

建议每个程序员没事可以到上面多逛逛，**多看优秀开源项目的代码和文档**，**尽量参与进去**，不论是 maillist、QQ / 钉钉群组等等，可以尝试从编写文档、写测试用例开始入手，逐步切入核心功能。

### 2、**Stack Overflow**

网站地址：[https://stackoverflow.com/](https://link.zhihu.com/?target=https%3A//stackoverflow.com/)

![](https://images.weserv.nl/?url=https%3A//pic2.zhimg.com/v2-af086c848bb9ee79c499ac384829e390_r.jpg%3Fsource%3D1940ef5c)

一个程序员问答网站，跟知乎一样，又跟知乎不一样。上面非常多的热心的大牛。

比如知乎虽然是个问答网站，但是你提问技术问题，答案只会告诉你关注 XXX 领取 Java 电子书、面试资料。

上面这句话纯属开玩笑。区别在哪里呢？**Stack Overflow 不欢迎任何讨论性的，非 Problem-Solving 类的问题**。什么意思呢？你可以提问 1+1 等于几，但是不要提问为什么要学 c 语言？

所以，如果你在开发过程中遇到了什么 bug，只要你搜索能力优秀，99% 在上面能找到答案。

在这里能够与很多有经验的开发者交流，如果你是有经验的开发者，还可以来这儿帮助别人解决问题，提升个人影响力。

（二）开源项目推荐
---------

参与优秀的开源项目可以和业界优秀的程序员大神一起结对编程，开阔思路，掌握优秀的代码编写风格、设计模式和技巧。

在此推荐几个业界有名的优秀的开源项目：

**1、Redis：[Redis](https://link.zhihu.com/?target=https%3A//redis.io/)**

Redis 的性能极高且拥有丰富的数据类型，同时，Redis 所有操作都是原子性的，也支持对几个操作合并后原子性的执行。另外，Redis 有丰富的扩展特性，它支持 publish/subscribe, 通知, key 过期等等特性。 Redis 更为优秀的地方在于，它的代码风格极其精简，整个源码只有 23000 行，很有利于阅读和赏析。

**2、RocketMQ：[Apache RocketMQ](https://link.zhihu.com/?target=https%3A//rocketmq.apache.org/)**

阿里巴巴开源的消息中间件，久经双 11 大促流量洪峰考验。

![](https://images.weserv.nl/?url=https%3A//pic2.zhimg.com/v2-b750564779cf839726ee544376520ef6_r.jpg%3Fsource%3D1940ef5c)

**3、Flink:[Stateful Computations over Data Streams](https://link.zhihu.com/?target=https%3A//flink.apache.org/)**

**Apache Flink** 是由 [Apache 软件基金会](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/Apache%25E8%25BD%25AF%25E4%25BB%25B6%25E5%259F%25BA%25E9%2587%2591%25E4%25BC%259A)开发的[开源](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/%25E5%25BC%2580%25E6%25BA%2590)流处理[框架](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/%25E8%25BD%25AF%25E4%25BB%25B6%25E6%25A1%2586%25E6%259E%25B6)，其核心是用 [Java](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/Java) 和 [Scala](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/Scala) 编写的分布式流数据流引擎。Flink 以[数据并行](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/%25E6%2595%25B0%25E6%258D%25AE%25E5%25B9%25B6%25E8%25A1%258C)和[管道](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/%25E7%25AE%25A1%25E9%2581%2593_%28%25E8%25BD%25AF%25E4%25BB%25B6%29)方式执行任意流数据程序，Flink 的流水线运行时系统可以执行[批处理](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/%25E6%2589%25B9%25E5%25A4%2584%25E7%2590%2586%25E4%25BB%25BB%25E5%258A%25A1)和流处理程序。此外，Flink 的运行时本身也支持[迭代算法](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/%25E8%25BF%25AD%25E4%25BB%25A3%25E6%25B3%2595)的执行。

Flink 提供高吞吐量、低延迟的流数据引擎以及对事件 - 时间处理和状态管理的支持。Flink 应用程序在发生机器故障时具有容错能力，并且支持 exactly-once 语义。程序可以用 Java、Scala、[Python](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/Python) 和 [SQL](https://link.zhihu.com/?target=https%3A//zh.wikipedia.org/wiki/SQL) 等语言编写，并自动编译和到在集群或云环境中运行的数据流程序。

![](data:image/svg+xml;utf8,<svg xmlns='http://www.w3.org/2000/svg' width='370' height='188'></svg>)

**6、RxJava**：java 平台的 Reactive 扩展。

第二步：这个项目有什么技术难点，用了什么解决方案。实现了什么需求。（抓主干）

![](https://images.weserv.nl/?url=https%3A//pic1.zhimg.com/v2-44491e4821f2526b01ba4e2398c438a5_r.jpg%3Fsource%3D1940ef5c)

第五步：加功能，其实只要你扒下来项目，加几个简单功能的过程都是你独立思考的过程。

4\. 代码花金币下载。一开始觉得这个挺好的，别人写好了一些模块代码，需要花金币去下载，也是对别人的付出一种肯定。而且 CSDN 还让自己发布文章也赚涨金币，想下代码，那就多写点文章输出，还能激励自己。但自从我下了几回文章后面的代码，发现写的都是什么啊，跟文章里说的全然不一样，乱七八糟的，都不知道写了点啥，根本没法参考，更别说用，当然这跟社区氛围有关，需要社区做好引导，加大惩罚，让用户珍惜自己账号，提升共同维护好社区的意识。
